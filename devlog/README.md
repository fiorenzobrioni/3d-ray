# devlog - 3D-Ray

Cartella dei log di sviluppo e note di design. Per roadmap, TODO, bug noti e checklist vedi [`PLANNING.md`](../PLANNING.md).

## Struttura

- `CURRENT.md` - log attivo, sviluppi piu recenti in cima
- `YYYY/YYYY-MM-DD.md` - log storici archiviati, organizzati per anno di contenuto

## Regole di archiviazione

Il log attivo (`CURRENT.md`) viene archiviato quando:

- supera circa 800 righe, oppure
- si inserisce il primo sviluppo di un anno diverso da quello indicato in `CURRENT.md` (il log viene archiviato prima dell'inserimento)

**Come archiviare:** spostare `CURRENT.md` in `YYYY/YYYY-MM-DD.md`, dove `YYYY` e l'anno indicato nell'intestazione del file e `YYYY-MM-DD` e la data odierna. Creare un nuovo `CURRENT.md` vuoto con l'anno corrente in intestazione.
